📚 How These Attacks Work

Cross-Site Scripting (XSS)

XSS occurs when an attacker injects malicious scripts into web pages viewed by other users. The injected code executes in victims' browsers, potentially stealing session cookies, redirecting users, or capturing keystrokes.

Reflected XSS Example
<script>alert('XSS')</script>
<img src=x onerror=alert(1)>
// URL: ?name=<script>...</script>

SQL Injection (SQLi)

SQL Injection occurs when user-supplied data is inserted directly into SQL queries without proper escaping. Attackers can bypass authentication, extract sensitive data, modify records, or drop entire tables.

Classic SQLi Example
-- Input: admin' OR '1'='1
SELECT * FROM users
WHERE user='admin' OR '1'='1'
-- Always TRUE → bypasses login!

Defense Strategies

🛡️
Output Encoding
HTML-encode all user data before rendering. Use htmlspecialchars() in PHP, encodeURIComponent() in JS.
🔒
Prepared Statements
Use parameterized queries with PDO or MySQLi. Never concatenate user input into SQL strings.
📋
Content Security Policy
Set CSP headers to restrict script sources and prevent inline script execution.
✅
Input Validation
Validate and whitelist input types. Reject unexpected characters server-side, not just client-side.
PHP Secure Prepared Statement
$stmt = $pdo->prepare(
  "SELECT * FROM users WHERE email = :email"
);
$stmt->execute([':email' => $email]);
